# 작업 내역

## 1. 라이브 세션 제목 프론트엔드 수신으로 변경

**변경 전:** 서버에서 `"라이브 회의 " + LocalDateTime.now()` 자동 생성  
**변경 후:** `POST /api/v1/live/sessions { "title": "..." }` 로 프론트에서 직접 전달

- `LiveController.java` — `POST /api/v1/live/sessions` 엔드포인트 추가
- `LiveWebSocketHandler.java` — `onOpen` 시 세션 생성 제거, WebSocket 경로 `/api/v1/live/ws/{meetingId}` 로 변경

---

## 2. Spring Boot 3.3 빌드 오류 수정

`RestTemplateBuilder.connectTimeout(Duration)` 메서드가 Spring Boot 3.3에서 제거됨

- `RestTemplateConfig.java` — `SimpleClientHttpRequestFactory` + int(ms) 방식으로 교체

---

## 3. LLM 중복 호출 / 요약 이중 저장 수정

Python LLM 서버가 같은 `meetingId`로 여러 번 저장되는 문제

- `summary_chain.py` — `integrated_audio_summary()`의 중복 `_save_summary_to_java()` 호출 제거
- `LlmService.java` — Python 호출 전 DB 캐시 확인 추가 (이미 `COMPLETED` 이면 LLM 스킵)
- `summary_chain.py` — `get_summary()` 에 Java `GET /api/v1/meetings/{id}` 캐시 확인 추가

---

## 4. 요약 내용 불일치 수정

같은 회의인데 상세 조회할 때마다 요약 내용이 달라지는 문제  
원인: 프론트가 Python `/api/summary` 를 직접 호출 → LLM 2번 실행 (temperature=0.3으로 결과가 달라짐)

- Python `get_summary()` 에 캐시 확인 로직 복원
- `GET /api/v1/meetings/{id}` 는 DB에서만 반환하도록 정리

---

## 5. DTO / 서비스 리팩토링

컴파일 오류 수정 과정에서 발생한 변경

- `MeetingDetailResponse.java` — 존재하지 않는 `SummaryDto.ActionItemDto` 참조 제거
- `TranscribeResponse.java` — `from()` 파라미터 `MeetingSummary` → `SummaryDto` 로 변경
- `LlmService.java` — `summarize()` 시그니처 `Meeting` → `UUID` 로 변경, 반환 타입 `MeetingSummary` → `SummaryDto`

---

## 6. 온라인 회의 참여자 / 권한 API 구현

팀 채팅 스펙 기반으로 8개 엔드포인트 구현

### 신규 파일

| 파일 | 설명 |
|------|------|
| `model/ParticipantRole.java` | ADMIN / GUEST enum |
| `model/MeetingParticipant.java` | meeting_participants 테이블 엔티티 |
| `repository/MeetingParticipantRepository.java` | JPA 레포지토리 |
| `service/OnlineMeetingService.java` | 비즈니스 로직 (생성·초대·참여·권한·퇴장·시작·종료) |
| `controller/OnlineMeetingController.java` | REST 컨트롤러 |
| `dto/CreateMeetingRequest.java` | 회의 생성 요청 DTO |
| `dto/InviteResponse.java` | 초대 토큰 응답 DTO |
| `dto/ParticipantResponse.java` | 참여자 응답 DTO |
| `dto/UpdateRoleRequest.java` | 역할/권한 변경 요청 DTO |

### 기존 파일 수정

| 파일 | 변경 내용 |
|------|-----------|
| `model/MeetingType.java` | `DISCORD` 추가 |
| `model/Meeting.java` | `invite_token` 컬럼 필드 추가 |
| `core/exception/Exceptions.java` | `AccessDeniedError` (EX-013, 403) 추가 |

### 엔드포인트 목록

| Method | Path | 설명 |
|--------|------|------|
| `POST` | `/api/v1/meetings` | 온라인 회의 생성 (ADMIN 자동 등록) |
| `POST` | `/api/v1/meetings/{id}/invite` | 초대 토큰 생성 |
| `GET` | `/api/v1/meetings/{id}/join?token=` | 토큰으로 참여 |
| `GET` | `/api/v1/meetings/{id}/participants` | 참여자 목록 |
| `PATCH` | `/api/v1/meetings/{id}/participants/{profileId}/role` | 역할/권한 변경 |
| `DELETE` | `/api/v1/meetings/{id}/participants/{profileId}` | 참여자 퇴장/강퇴 |
| `POST` | `/api/v1/meetings/{id}/start` | 회의 시작 → status LIVE |
| `POST` | `/api/v1/meetings/{id}/end` | 회의 종료 → status COMPLETED |

> 모든 엔드포인트는 `X-User-Id` 헤더로 사용자(profile_id) 식별
